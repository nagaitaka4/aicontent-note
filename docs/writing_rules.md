# ライティングルール

## 前提
本ルールはブログ記事（長文コンテンツ）を前提とする。

## 基本方針
・現実的であること
・誇張しない
・ワクワク感を残す
・読みやすい
・自虐を入れる
・ネガティブな表現ばかりは避ける

## NG
・煽り
・抽象論
・誇張
・再現性のない話
・AI万能論

## OK
・実体験
・具体例
・数字
・検証結果
・失敗談
・成功談
・リアルな感想
・現実的な未来予測
・実務に落とす方法

## 構成ルール
H1：タイトル（必ず記事の内容を表すもの）
H2：H1の問題提起/結論などの主要トピック
H3：具体/詳細解説（必要な場合のみ）の小見出し

## トーン
・丁寧
・フラット
・信頼感重視
・親しみやすい
・専門用語は必要に応じて説明する

## 構造
導入（見出しなし）

H2
H3（必要な場合のみ）

結論

## 文体
・シンプル
・論理的
・短文
・わかりやすく
・会話調も適度に入れる

## 文体詳細ルール（no:7を基準とする）

no:7「コンテンツ運用が続かない理由と、止めないための仕組みの作り方」の文体・トーン・改行を基準にする。

### 一人称
・「私」ではなく「自分」を使う
・「自分にもありました」「自分の体験を振り返ってみると」のような使い方が自然

### 改行リズム
・句点（。）を基準に改行する
・読点（、）では改行しない
・短い文は1行にまとめる。長い文は句点で区切って改行する
・「〇〇です。△△です。」の短文連続でリズムを作る

例（OK）：
```
テーマを決めて、最初の1本を書いて、次も書いて。
そのはずが、気づけば更新が止まっている。
このパターンを何度か経験してから、これは意志の問題ではないと感じるようになりました。
```

例（NG：読点で細かく切る）：
```
テーマを決めて、
最初の1本を書いて、
次も書いて。
```

MDファイルでの改行はWordPressでスペースとして表示されるため、段落の区切りには空行を入れる。

### 断定の避け方
強い断定は使わない。以下の表現で余白を持たせる。
・「〜のような気がしています」
・「〜に近いと思います」
・「〜がほとんどです」
・「〜のだと思います」
・「〜かもしれません」

### 本音を出す表現
・「正直に言うと〜」
・「やっぱり〜」
・「今思えば〜」
・「無理やり自分に言い聞かせていましたが」

### 軽い自虐・自己ツッコミ
過度にならない範囲で入れる。読者が「あ、自分もそうだ」と思える。
・「やる気満々だったからです。」（短文でオチ）
・「あの時間はいったい何だったのか・・・。」
・「・・・」を余韻・間として使ってよい

### 読者への語りかけ方
・「〜しましょう」の指示形は使わない
・「〜と感じている方は」「〜のケースが多いように感じます」など、柔らかく寄り添う形
・命令ではなく、体験の共有として書く

### H3見出しの作り方
読者の感情・状況を言語化する。「あ、それ自分だ」と思えるものにする。
・「〜は理由のように見えて、実は結果の話」
・「〜ではなく、〜がなかった」
・断定より「気づき」のトーンで

### 箇条書きの使い方
・多用しない。本文の流れで書けるなら文章にする
・構造を整理するときだけ使う（3つの変化・パターン比較など）

### 記事の流れ（H2の設計）
① 自分の体験・失敗から入る（共感の入口）
② なぜそうなるかを構造で説明する（読者が「自分だけじゃない」と感じる）
③ 自分が変えたことを書く（解決策。「やれ」ではなく「自分はこうした」）
④ 具体的な現在のフローを書く（再現性）
⑤ まとめ・読者へのメッセージ（押しつけない）

## シリーズ記事のルール

### H3の追加タイミング
- H2セクション内の段落が4つ以上になる場合、H3を1〜2個追加して読みやすくする
- 話題の転換点（「もう一つ」「次に」など）がH3の目安
- H3タイトルは読者の感情・状況を言語化する（「あ、それ自分だ」と思えるもの）

### サイト内リンク（前回記事へのリンク）
- シリーズ記事（side-businessなど）の導入に、前回記事へのサイト内リンクを自然な文脈で入れる
- リンク単体を行として置くのではなく、「前回〇〇をした」「〇〇まで試みた」など前回の内容を軽く振り返る文章の中にリンクを埋め込む
- 例：`[前回](URL)、〇〇を試みました。結果は〇〇で、そこから今回の〇〇につながります。`

### CTAの扱い
- **副業検証シリーズ（side-business）にはCTAを入れない**
  - 読者層が「運用代行の検討者」ではなく「AIライティングに興味がある人」のため、導線として不自然
  - 信頼構築（間接的）を目的とする記事のため、CTAで水を差さない
- CTAを入れるのはSEO記事・仕事記事（J系）のみ
